POST /persistence/loadsession
Load / reactivate session

Reactivates a session so the owner can open it again via the accessURL (/index link). The access link is consumed on first use — after the owner opens the session, this endpoint must be called again to generate a fresh link. This is only needed while the session is not in external mode.
