inSign API - Getting Started

Integrate electronic signatures into your application in minutes.

This guide is for developers and anyone interested in exploring the inSign API.
Try it out right here - completely free, no registration needed.

Not looking for technical details? Visit www.getinsign.com for product information.

Swagger API Docs Full API Explorer getinsign.com
1
Create Session
POST /configure/session

Upload a document and create a signing session. The response contains a sessionid.
You can provide your PDF in several ways: as a public fileURL (used below), as a Base64-encoded string via fileB64, or as a multipart/form-data upload. For details see the Swagger docs.

2
Begin Signing
POST /extern/beginmulti

Invite external participants to sign the document. Each signer is assigned a role matching the signature fields in the PDF.
You control how signers access the session: set sendEmails: true to have inSign send invitation emails automatically, or use singleSignOnEnabled: true to get back a direct URL you can embed in your own app or workflow. Signers can be added inOrder (sequential signing) or all at once (parallel). Each signer only sees the fields assigned to their role.

3
Check Status
POST /get/status

Poll the session status to see signing progress, document metadata, and signature field details.

4
Download Document
POST /get/document

Download the signed PDF. Parameters are passed as query string.

Java API Library

For Java projects, the insign-java-api library provides a typed client with builders and configuration helpers.

<dependency>
  <groupId>com.getinsign</groupId>
  <artifactId>insign-java-api</artifactId>
</dependency>

Add the GitHub Package Registry to your Maven settings.xml - see Working with the Apache Maven registry.
